1. 核心方向
遊戲視角

採用 2.5D 俯視角。

實際素材全部使用 2D Sprite，但透過：

攝影機角度
角色 Sprite 排列
Y 軸排序
地圖物件遮擋

來營造類似 2.5D 的空間感。

參考感覺：

《進擊羔羊傳說》那種可愛、平面角色，但場景有俯視深度的視覺。

2. 技術前提
Unity 系統

使用：

Unity 2D 專案
New Input System
Rigidbody2D
Collider2D
Cinemachine Camera，可選
Sprite Renderer
Tilemap 或大型 Sprite 地圖皆可
3. 玩家移動需求
操作方式

玩家可以使用：

操作	功能
W / ↑	往上移動
S / ↓	往下移動
A / ←	往左移動
D / →	往右移動

之後可支援手把，但 Phase 0.1 先不做。

移動規則

玩家必須能：

上下左右自由移動
斜向移動時速度不變快
不會穿牆
不會離開地圖邊界
移動時角色有基本方向感

Phase 0.1 可以先不做完整動畫，只要能確認角色方向與移動手感即可。

4. Input System 設計

建立一個 Input Action Asset：

Action Map：Player
Action	Type	Binding
Move	Value / Vector2	WASD
Move	Value / Vector2	Arrow Keys

輸出 Vector2：

x = 左右方向
y = 上下方向

移動邏輯由 PlayerMovement 接收。

5. Player 架構

玩家物件建議結構：

Player
├─ SpriteRenderer
├─ Rigidbody2D
├─ CapsuleCollider2D 或 CircleCollider2D
├─ PlayerInput
└─ PlayerMovement.cs
Rigidbody2D 設定
項目	設定
Body Type	Dynamic
Gravity Scale	0
Freeze Rotation Z	true
Collision Detection	Continuous 或 Discrete
Interpolate	Interpolate
6. PlayerMovement 設計
職責

PlayerMovement 只負責：

接收 Move Input
計算移動方向
控制 Rigidbody2D 移動
更新角色面向

不處理攻擊、互動、升級等其他功能。

移動流程
讀取 Move Input
↓
Normalize 方向，避免斜向加速
↓
FixedUpdate 使用 Rigidbody2D 移動
↓
根據最後移動方向更新角色面向
7. 2.5D 視覺設計
攝影機

使用 Orthographic Camera。

建議設定：

項目	建議
Projection	Orthographic
Rotation	可先維持 0,0,0
Camera Angle	先用俯視 2D，之後再微調假 2.5D
Size	6～8 之間測試
Follow	玩家

Phase 0.1 重點不是精緻視角，而是先確認：

這套 2D 素材能不能用俯視角移動起來。

場景深度

用 Y 軸排序 做前後遮擋。

規則：

畫面越下方的物件，顯示在越前面
畫面越上方的物件，顯示在越後面

例如：

玩家走到樹後面時，被樹擋住
玩家走到樹前面時，蓋在樹前面
8. 地圖設計

Phase 0.1 需要一張簡單大地圖。

地圖內容：

一塊可走區域
一些不可穿越障礙物
基本邊界
幾個裝飾物件測試遮擋

例如：

大地圖
├─ 地面
├─ 樹 / 蘑菇 / 石頭
├─ 牆壁或邊界
└─ 玩家出生點
9. 碰撞設計
玩家碰撞

玩家需要 Collider2D。

建議使用：

CircleCollider2D：手感較滑順
CapsuleCollider2D：角色比較直立時可用
地圖碰撞

障礙物需要 Collider2D。

例如：

石頭
樹幹
牆壁
地圖邊界

Phase 0.1 不需要複雜碰撞，只要玩家不能穿過主要障礙即可。

10. 攝影機跟隨

攝影機要跟著玩家移動。

優先做法：

Cinemachine Virtual Camera Follow Player

最低需求：

玩家移動時鏡頭跟著走
鏡頭不要抖動
不需要做邊界限制

如果有時間，再加 Camera Boundary。

11. 驗收標準

Phase 0.1 完成條件：

玩家可以用 WASD / 方向鍵移動
斜向移動速度正常
玩家不會穿過障礙物
玩家不會掉出地圖外
攝影機會跟隨玩家
2D 素材在畫面中有基本 2.5D 俯視感
物件有基本前後遮擋感
12. 暫時不做

Phase 0.1 不包含：

戰鬥
對話
技能
升級
敵人
UI
音效
任務
複雜動畫
地圖切換
自動生成地圖
Phase 0.1 一句話版本

先做一個 2D 素材 + 2.5D 俯視攝影機 + New Input System 移動 + 大地圖碰撞 + 鏡頭跟隨 的基本移動底座，確認玩家可以在類似《進擊羔羊傳說》的場景中順暢上下左右移動。
